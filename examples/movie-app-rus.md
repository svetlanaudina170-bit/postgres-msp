## Веб-сайт с рейтингами фильмов

Давайте проведём быструю AI-сессию по кодингу и превратим идею в работающий проект!

Мы будем использовать [датасет IMDB](https://developer.imdb.com/non-commercial-datasets/), чтобы создать веб-сайт с рейтингами фильмов.

**Наши AI-инструменты:**

* **Replit** — для первоначального прототипа
* **Cursor** — как AI-агент для программирования
* **Postgres Pro** — чтобы предоставить Cursor эксперта по Postgres

**Что мы сделали:**

1. Создали первоначальное приложение на Replit — оно медленное!
2. Оптимизировали производительность — включая ORM-запросы, индексацию и кэширование
3. Исправили пустые страницы с деталями фильма
4. Улучшили сортировку лучших фильмов по рейтингу

**Начнём...**

### 1) Создание первоначального приложения на Replit

Мы задали Replit следующий промпт:

> Создай веб-приложение на основе Flask, Python и SQLAlchemy ORM
> Это веб-сайт, использующий схему из публичного датасета IMDB. Предположим, что датасет IMDB импортирован как есть, и дополни его. Я хочу, чтобы пользователи могли просматривать адаптированную под мобильные устройства страницу для каждого фильма, со всеми связанными данными IMDB. Дополнительно, пользователи могут ставить фильму оценку от 1 до 5 и просматривать фильмы с наивысшим рейтингом. Сообщество и эти оценки — одна из ключевых целей сайта.

**Бах!**
У нас есть полностью функциональный сайт с рейтингами, поиском, просмотром, авторизацией — меньше чем за час. Что?!

**Но он оооочень медленный...**
AI-агент создал кучу кода на ORM и, похоже, добавил какие-то индексы, но они были неудачны.

### 2) Оптимизация запросов

Сайт выглядит прилично, но слишком медленный, чтобы его запускать.
Переключаемся на Cursor с Postgres Pro, чтобы подготовить приложение к публикации.

Промпт:

> Моё приложение медленное!
>
> * Ищи способы ускорить его, улучшив запросы, индексы или кэширование.
> * Для изменений в базе используй миграционные скрипты, которые я смогу применить позже.

**Что сделал AI-агент:**

1. Изучил схему и код, чтобы найти потенциально проблемные запросы
2. Использовал Postgres Pro для диагностики: `get_top_queries`, `analyze_db_health`, `analyze_query_indexes`
3. Добавил несколько индексов для ускорения запросов
4. Удалил неиспользуемые и раздутые индексы для освобождения места
5. Добавил кэширование для тяжёлых запросов и загрузки изображений
6. Создал миграционный скрипт для применения изменений

**Это было круто!**
Агент связал анализ базы с кодом и подготовил полноценный pull request за 2.5 минуты.

**Ожидаемый эффект:**

* Текстовый поиск станет быстрее в 10–100 раз
* Загрузка страниц — в 2–5 раз быстрее
* Нагрузка на базу данных существенно уменьшится
* Внешние API-вызовы сократятся примерно на 90%

### 3) Исправление пустых страниц деталей фильма

Страница фильма выглядит пустой. Давайте разберёмся.

Промпт:

> Страница с деталями фильма выглядит ужасно.
>
> * нет актёров/съёмочной группы. У нас нет данных или запрос неверный?
> * Рейтинг стоит в неподходящем месте. Перемести его ближе к заголовку.
> * Есть ли дополнительные данные, которые мы можем отобразить, вроде описания? Проверь схему.

**Результат:**

1. Агент использовал Postgres Pro для анализа схемы и сравнения с кодом
2. Исправил запрос в маршруте, добавив `join` с `name_basics`
3. Нашёл дополнительные данные в `title_basics` и создал новый раздел «О фильме» с жанром, длительностью и годом выпуска

Задаём вопрос:

> А какие данные мы вообще пропустили?

AI-агент запускает SQL-запросы и выясняет, что действительно отсутствуют данные о съёмочной группе.
Он пишет надёжный скрипт для импорта данных заново.
*(оказалось, первоначальный скрипт завершался с ошибкой и не догружал данные)*

### 4) Улучшение сортировки фильмов с высоким рейтингом

Страница топа показывает странные фильмы вроде "Sisters of the Shrink 4" и "Zhuchok" — что-то не так!

Промпт:

> Как сортируются фильмы с высоким рейтингом? Похоже, случайно.
> У нас есть данные в таблицах? Запрос работает правильно?

AI-агент проверяет данные и код и выясняет, что проблема в отсутствии ограничения по `num_votes`.

Задаём:

> Помоги определить адекватный минимум отзывов

AI-агент анализирует распределение по данным и примерам и предлагает порог в 10 000 голосов.
Результаты обоснованы, а не взяты из головы.
